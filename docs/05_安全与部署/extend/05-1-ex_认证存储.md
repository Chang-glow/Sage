# 05-1-ex 认证存储

## 1. 概述

本文档是  [05-1_API安全](../05-1_API安全.md)  的补充，定义邀请码和后台密码的生成、存储与验证方式。

## 2. 安全原则

凭据使用  .env  文件存储，该文件不提交 Git（已在  .gitignore  中排除）。在本地运行的单用户环境下，操作系统文件权限提供足够的安全性。

## 3. 邀请码

### 3.1 生成

项目提供脚本自动生成随机邀请码：

```
python scripts/generate_secrets.py --invite
```

脚本使用  secrets.token_urlsafe(8)  生成强随机邀请码，输出到终端并自动写入  .env  文件。

如需多个邀请码，以逗号分隔存储在  INVITE_CODE  环境变量中。

### 3.2 验证

验证时比对用户提交的邀请码与  .env  中的值，一致即通过。

## 4. 后台密码

### 4.1 生成

```
python scripts/generate_secrets.py --admin
```

脚本使用  secrets.token_urlsafe(12)  生成强随机密码，输出到终端并自动写入凭据文件，同时使用  bcrypt  哈希存储。

### 4.2 首次部署流程

系统首次启动时，若检测到密码仍为默认值  admin ，则在 Web UI 上拦截登录流程：

1. 管理员访问后台，使用默认账密  admin  /  admin  登录
2. 系统检测到初始密码，跳转至密码修改页面
3. 输入新密码并确认，提交后系统使用  bcrypt  哈希存储
4. 修改完成后自动进入后台

新密码必须为强密码（长度 ≥ 8，含大小写字母和数字）。在未完成修改前，无法访问后台任何功能。

### 4.3 密码修改

管理员可在后台设置页面随时修改密码，流程同首次部署。
