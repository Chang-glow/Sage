# 05-4 数据库备份

## 1. 概述

PostgreSQL 数据库存储夕照雅巷的全部社区数据。备份策略需兼顾数据安全与运维简便，支持本地备份和异地同步。所有备份操作通过脚本自动执行。

## 2. 备份策略

| 备份类型 | 频率 | 保留策略 | 用途 |
|---------|------|---------|------|
| 每日增量备份 | 每天 1 次 | 保留最近 7 天 | 快速恢复近期数据 |
| 每周全量备份 | 每周 1 次 | 保留最近 4 周 | 完整恢复点 |
| 手动备份 | 按需 | 永久保留 | 重大更新前 |


## 3. 备份命令

### 3.1 全量备份

```
pg_dump -U username -h localhost -F c -b -v -f "backup/sga_$(date +%Y%m%d).dump" sga
```

- -F c：自定义压缩格式
- -b：包含大对象
- 输出为单文件，压缩后通常 < 20 MB

### 3.2 增量备份

PostgreSQL 通过 WAL（Write-Ahead Log）归档实现增量备份。配置 postgresql.conf：

```
wal_level = replica
archive_mode = on
archive_command = 'cp %p /backup/wal/%f'
```

## 4. 自动化脚本

提供备份脚本 scripts/backup.sh：

```
#!/bin/bash
BACKUP_DIR="./backups"
DATE=$(date +%Y%m%d)
pg_dump -U sga_user -h localhost -F c -b -v -f "$BACKUP_DIR/sga_$DATE.dump" sga
# 删除 7 天前的备份
find "$BACKUP_DIR" -name "*.dump" -mtime +7 -delete
```

通过 cron 定时执行：

```
0 4 * * * /path/to/sage/scripts/backup.sh
```

## 5. 异地同步（可选）

使用 rclone 将备份文件同步到云存储：

```
rclone copy ./backups remote:sga-backups/
```

支持的免费云存储：

- OneDrive（5GB 免费）
- Google Drive（15GB 免费）
- 阿里云盘（通过第三方 rclone 插件）

## 6. 恢复流程

### 6.1 从全量备份恢复

```
pg_restore -U username -h localhost -d sga -v backup/sga_20260514.dump
```

### 6.2 恢复到特定时间点

结合全量备份和 WAL 归档，可恢复到任意时间点：

```
pg_restore -U username -h localhost -d sga -v --recovery-target-time="2026-05-14 12:00:00" backup/sga_20260514.dump
```

## 7. 数据量预估

年度原始数据约 400-800 MB，压缩后约 10-20 MB/次全量备份。保留 7 天增量 + 4 周全量，总备份占用 < 200 MB。

## 8. 监控

- 每次备份后检查脚本执行状态
- 定期测试备份文件可恢复性（每月一次，手动执行）
- 备份失败时发送日志告警
