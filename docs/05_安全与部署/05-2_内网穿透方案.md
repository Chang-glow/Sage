# 05-2 内网穿透方案

## 1. 概述

夕照雅巷的后端运行在本地，需要通过内网穿透让朋友在外网访问。本文档提供几种可选方案，按使用场景推荐。

## 2. 方案一：Cloudflare Quick Tunnels（临时分享）

### 2.1 适用场景

- 临时给朋友演示
- 不需要固定域名
- 零配置、免注册

### 2.2 使用方式

安装 cloudflared 后，一行命令启动：

```
cloudflared tunnel --url http://localhost:19175
```

- 自动生成 https://随机字符串.trycloudflare.com 域名
- 关闭命令行窗口即销毁隧道
- 完全免费，不需要注册 Cloudflare 账户
- 自带 HTTPS

### 2.3 限制

- 域名随机生成，每次重启变化
- 无法绑定自定义域名
- 无 SLA 保证

## 3. 方案二：frp（推荐，固定分享）

### 3.1 适用场景

- 需要固定访问地址
- 有一台带公网 IP 的服务器（借用或购买轻量云服务器）
- 想绑定自己的域名

### 3.2 架构

- 服务端（frps）：部署在公网服务器上，开放一个端口
- 客户端（frpc）：部署在本地机器上，将 localhost:19175 映射到服务端端口

### 3.3 配置示例

服务端 frps.toml：

```
bindPort = 7000
```

客户端 frpc.toml：

```
serverAddr = "公网服务器IP"
serverPort = 7000

[[proxies]]
name = "sage-web"
type = "tcp"
localIP = "127.0.0.1"
localPort = 19175
remotePort = 8080
```

朋友访问 http://公网服务器IP:8080 即可。

### 3.4 绑定域名

- 在域名 DNS 中添加 A 记录，指向公网服务器 IP
- 在服务器上用 Nginx 反向代理 frps 端口
- 朋友访问 https://你的域名 即可

## 4. 方案三：Tailscale Funnel（备选）

### 4.1 适用场景

- 你与朋友都使用 Tailscale
- 需要加密通信

### 4.2 使用方式

在本地启用 Tailscale Funnel：

```
tailscale funnel 19175
```

- 生成公网链接
- 免费版有一定流量限制
- 国内网络环境下可能不稳定

## 5. 方案对比


| 方案 | 固定域名 | 配置难度 | 需要额外服务器 | 适合场景 |
|------|---------|---------|-------------|---------|
| Quick Tunnels | 否 | 极低 | 否 | 临时演示 |
| frp | 是 | 中 | 是 | 长期分享 |
| Tailscale Funnel | 否 | 低 | 否 | 已有 Tailscale 网络 |


## 6. 安全注意事项

无论使用哪种方案：

- 所有公网流量都经过  [05-1_API安全](./05-1_API安全.md)  中定义的 Nginx 速率限制
- 后端 API 必须保持认证和速率限制
- 邀请码机制仍然生效
- 不直接暴露 FastAPI 端口（19175），仅暴露 Nginx 代理端口
