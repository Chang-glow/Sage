# 05-3 前端部署

## 1. 概述

夕照雅巷的前端为纯静态页面，部署于 Cloudflare Pages。通过绑定自定义域名获得稳定访问地址，自动启用 HTTPS 和全球 CDN 加速。

## 2. 部署平台

### 2.1 Cloudflare Pages

- 免费计划：无限带宽、无限请求、每月 500 次构建
- 自动 HTTPS，全球 CDN
- 支持自定义域名
- 注册和使用不需要信用卡

### 2.2 备选方案

- Vercel / Netlify：功能类似，免费额度各有千秋
- GitHub Pages：零学习成本但国内访问不稳定

## 3. 部署流程

### 3.1 构建命令

前端项目构建为静态文件：

```
npm run build
```

构建产物输出到 dist/ 目录。

### 3.2 关联 Git 仓库

1. 将前端代码推送到 GitHub 仓库
2. 在 Cloudflare Pages 控制台连接该仓库
3. 设置构建命令：npm run build 
4. 设置输出目录：dist 
5. 每次推送自动触发构建和部署

### 3.3 环境变量

在 Cloudflare Pages 控制台设置以下环境变量：


| 变量名 | 说明 | 示例 |
|--------|------|------|
| VITE_API_BASE_URL | 后端 API 地址 | https://your-backend-domain.com |


## 4. 绑定自定义域名

### 4.1 前提

域名已托管在 Cloudflare DNS（或将 DNS 服务器修改为 Cloudflare 提供的地址）。

### 4.2 绑定步骤

1. 在 Cloudflare Pages 项目设置 → 自定义域
2. 输入你的域名（如 sga.yourdomain.com）
3. Cloudflare 自动配置 DNS 和 SSL 证书
4. 等待几分钟后，通过自定义域名访问前端

## 5. 安全特性

Cloudflare Pages 自动提供：

- DDoS 防护（不限规模）
- HTTPS 强制启用
- 基础 Web 应用防火墙（WAF）
- 全球 CDN 分担流量

前端页面中所有 API 请求均指向后端域名，后端的安全防护见  [05-1_API安全](./05-1_API安全.md) 。

## 6. 与后端通信

- 前端通过 VITE_API_BASE_URL 环境变量获取后端地址
- 所有 API 请求携带必要的认证信息（邀请码、替身标识等）
- 跨域请求由后端 CORS 配置允许前端域名
