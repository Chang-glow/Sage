# 01-2 地点分配逻辑

## 1. 概述

本文档定义 Agent 注册时如何根据硬条件（年龄、职业等）自动分配平陵市的具体地点，包括教育机构（学生）或工作单位（非学生），以及对应的居住区域。

分配过程在注册流程的“硬条件生成阶段”执行，完全由代码规则完成，不调用 LLM。所有地点数据来源于 [01-1_平陵市设定](./01-1_平陵市设定.md) 中定义的资源库。

---

## 2. 分配时机与输入

- 时机：Agent 硬条件生成之后、AI 自主选择兴趣之前。
- 输入：
  - age：整数，14-50
  - occupation_category：职业大类（如“学生”“普工”“文员”“个体户”“自由职业”等）
  - education_level（可选）：由年龄推导的学历层次
- 输出：
  - school_or_company：具体机构名称与 ID
  - district：居住区域名称与 ID
  - boarding（仅学生）：是否住宿，影响上线时段和日常约束

---

## 3. 分配规则

### 3.1 学生分配
（age < 18 或 occupation_category == "学生"）

#### 根据年龄匹配教育机构类型：

年龄区间 |教育机构类型 |对应资源库 ID
---|---|---
14-15 |初中|（当前资源库暂无，可预留） |-
15-18 |高中 / 职高 / 中专 |EDU-001 ~ EDU-005（高中）EDU-003（职高）EDU-009（工业学校）EDU-010（卫校）
18-22 |本科 / 专科 |EDU-006（文理学院）EDU-007（师范学院）EDU-008（职业技术学院）

#### 高中阶段分流与概率分配：

模拟现实中的“五五分流”（普高与职高/中专的比例大致相当，但不严格），并为每所学校设置录取概率，避免大量学生集中到同一所学校。概率权重可配置。

流向 |机构 ID |机构名称 |权重 |录取概率 |备注
---|---|---|---|---|---
普通高中 |EDU-001 |平陵一中 |5 |~12.5% |省重点，分数线最高
普通高中 |EDU-002 |平陵二中 |10 |~25% |普通高中，招生最多
普通高中 |EDU-004 |平陵四中 |6 |~15% |郊区普高
普通高中 |EDU-005 |平陵五中 |4 |~10% |新建普高
职业/中专 |EDU-003 |平陵三中 |8 |~20% |职高，招生量大
职业/中专 |EDU-009 |平陵工业学校 |4 |~10% |中专
职业/中专 |EDU-010 |平陵卫校 |3 |~7.5% |中专

#### 分流逻辑：

- 系统首先根据权重随机决定 Agent 流向“普通高中”或“职业/中专”。
- 在确定的流向内，再根据内部权重随机选择具体学校。
- 权重可在配置文件中调整，以改变各学校的学生比例。

#### 住宿/走读分配：

- 根据机构的 boarding 属性决定居住方式：
  - boarding: true → 住宿，居住区域为学校所在区域（大学城片区或机构附近）。
  - boarding: false → 走读，居住区域从老城区或新城区中随机选取（代表家庭住址）。
  - boarding: mixed → 随机 50% 概率走读或住宿。

示例：

- 17岁 → 流向普高 → 随机到 EDU-002（平陵二中，mixed）→ 随机为走读 → 居住区域随机为 RES-002（新城区）。
- 16岁 → 流向职中 → 随机到 EDU-009（平陵工业学校，boarding: true）→ 自动住宿 → 居住区域为 RES-004（厂区宿舍，因工业学校与工厂区相邻）。

### 3.2 非学生分配
（age >= 18 且 occupation_category != "学生"）

#### 根据职业大类匹配企业/单位。

以下为示例映射表，完整列表见 world/companies.yaml：

职业大类 |示例匹配机构 |居住区域推导
---|---|---
普工 |COM-001（电子厂）、COM-002（纺织厂） 厂区宿舍 |RES-004
文员/销售/会计 |COM-003（天元汽配）、COM-005（鑫旺超市）等 |新城区 RES-002 或老城区 RES-001
个体户 |COM-004（商贸城） |老城区 RES-001
外卖员/快递员 |COM-008（美团）、COM-010（顺丰） |老城区 RES-001 或开发区 RES-005
网约车司机 |COM-009（滴滴） |随机
医护人员 |COM-007（社区卫生服务中心） |新城区 RES-002
自由职业 |无固定单位 |随机（老城区或新城区）

#### 居住区域推导：

- 若职业有明确集中工作地点（如电子厂），则居住区域优先为附近的厂区宿舍或该区域。
- 若职业无固定工作地点，则随机分配。

示例：

- 28岁，职业为普工 → 随机到 COM-001（平陵电子厂）→ 居住区域 RES-004（厂区宿舍）。
- 35岁，职业为个体户 → 随机到 COM-004（商贸城）→ 居住区域 RES-001（老城区）。

---

## 4. 特殊情况处理

- 年龄在 18-22 岁但已非学生：按非学生分配，职业池包含普工、外卖员等。
- 职高/中专学生同时有实习：可分配至对口企业，但初期简化为全部按学生处理。
- 无匹配机构：若年龄在 14-15 且无初中资源库，暂归入高中资源库，标记 is_temporary: true，待补充初中资源后调整。

---

## 5. 数据来源与扩展

所有机构数据从 world/education.yaml 和 world/companies.yaml 加载。分配逻辑硬编码于 app/world/location_assigner.py，学校的权重、分流比例等参数可在配置文件中调整，新增机构时只需更新 YAML 文件，无需改代码。
