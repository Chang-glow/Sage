## 5. 硬编码公式特征分析

### 5.1 经验曲线: `f(n) = 100 * n^1.6`

- **类型**: 幂函数 (power function), 非对数
- **增长**: n=1→100, n=5→917, n=10→3360, n=14→6060
- **导数**: `f'(n) = 160 * n^0.6`，增长率持续上升
- **文档要求**: "对数特征"（增长越来越慢）
- **实际效果**: 符合"前快后慢"吗？初始确实快（Lv2 只需 100 XP），但严格来说幂函数 > 线性增长，只有指数 < 1 的幂函数才有"边际递减"效果
- **1.6 > 1**: 不是边际递减！每级需要的 XP **增量**在上升——Lv2→3 需要多 100 XP，Lv14→15 需要多约 6000 XP
- **结论**: 指数 1.6 的幂函数确实让"升级越来越难"，这是正确的设计意图（RPG 曲线），但文档说"对数"是术语误用。应该用指数 < 1 的幂函数（如 n^0.6）才能做到真正的"越往后越平缓"

### 5.2 过往经历: `f(age) = 10 * ln(age/18 + 1) + 10`

- **类型**: 对数函数
- **age=14**: 7.5→7 条
- **age=30**: 14.8→15 条
- **age=50**: 20.5→21 条
- **导数**: `f'(age) = 10/(age+18)`，年龄越大增量越小
- **效果**: 有边际递减，年龄翻倍（25→50）经历只增加约 40%
- **结论**: ✅ 良好设计

### 5.3 记忆淘汰评分: `score = importance / (1 + t/T)`

- **类型**: 调和衰减
- **当 t=0**: score = importance
- **当 t=T**: score = importance / 2
- **当 t→∞**: score → 0（永不为零）
- **效果**: 前半段衰减快（第一天衰减到 50% * importance），后半段慢
- **结论**: ✅ 符合"新旧更替"需求。等价于 Ebbinghaus 遗忘曲线的一阶近似

### 5.4 亲密度衰减: `decay = (t-7) * 0.01` (t>7)

- **类型**: 线性 ramp
- **7 天内**: 无衰减（缓冲期）
- **100 天**: -0.93（从 1.0 到 0.07）
- **问题**: 线性衰减没有"近因效应"——真实社交中，最近互动的人遗忘更快（指数衰减），线性衰减反而让久远关系越来越负（可能穿过零点进入负数）
- **结论**: ⚠️ 可以用，但指数衰减 `intimacy *= exp(-0.01*(t-7))` 更符合社交遗忘规律

### 5.5 post_hotness: 阶梯函数

```python
score = 0.3
if rc > 20: score += 0.3
elif rc > 10: score += 0.2
elif rc > 5: score += 0.1
# post with 21 replies == post with 999 replies
```

- **问题**: 无区分度——21 回复和 999 回复得分相同
- **建议**: `0.3 + 0.3 * (1 - exp(-rc / 20))`
  - rc=5: 0.47
  - rc=10: 0.52
  - rc=20: 0.57
  - rc=50: 0.60 (接近饱和)
  - rc=100: 0.60

### 5.6 自平衡惩罚: 过饱和 ×0.5

- **当 component 频率 > 70%**: 权重减半
- **效果**: 强力压制重复行为
- **问题**: 0.5 因子可能过于激进——从 dominate 直接砍半，可能导致行为剧烈波动
- **建议**: 改为 0.7（降 30% 而非 50%），或使用连续函数 `weight *= (1 - saturation)`
