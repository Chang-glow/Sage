# 06-9 伪富媒体生成

## 1. 概述

本文档定义  media_generation  Skill 的规范。该 Skill 负责将内容生成 Skill 输出的占位符统一转换为夕照雅巷的伪富媒体标签（ `[img: ...]` 、 `[emj: ...]` ），确保所有帖子、回复、总结帖中的伪媒体格式和风格一致。

## 2. 设计初衷

内容生成 Skill（回复、发帖、总结等）不需要学习伪富媒体标签的具体写法，只需用统一占位符表达“这里需要一张图/表情包”，由本 Skill 统一解析和格式化。调优时只改这一处。

## 3. 占位符语法

内容生成 Skill 在输出文本中插入以下占位符：

```
{{media: image, 描述文字}}
{{media: emoji, 描述文字}}
```

### 3.1 参数说明

- 第一参数：类型， `image`  或  `emoji`
- 第二参数：描述文字，自然语言，内容生成 Skill 按需填写

### 3.2 占位符示例

```
{{media: image, 一只橘猫蹲在窗台上吃猫条}}
{{media: emoji, 猫猫震惊.jpg}}
{{media: image, 放学路上的夕阳}}
{{media: emoji, 老年人竖起大拇指}}
{{media: image, 新买的多肉桃蛋放在书桌上}}
```

## 4. 处理流程

1. 内容生成 Skill（回复/发帖/总结等）输出带占位符的文本
2. 调度引擎在文本进入数据库前调用  media_generation  Skill
3. 本 Skill 解析文本中所有  {{media: ...}}  占位符
4. 对每个占位符，调用轻量决策模型将描述润色为统一风格的最终标签
5. 将占位符替换为最终标签，输出完整文本

## 5. 输出格式

### 5.1 图片标签

格式： `[img: 自然语言描述]`

要求：描述约 20-80 字，细节丰富但不啰嗦，用中性客观的语气补充视觉信息

转换示例：

输入占位符： {{media: image, 一只橘猫蹲在窗台上吃猫条}} 
`输出标签：` `[img: 一只橘猫蹲在老式铝合金窗台上，阳光打在半边脸上，正低头专心吃一根猫条]` 

### 5.2 表情包标签

格式： [emj: 简短描述] ``

要求：描述约 2-10 字，口语化，捕捉表情包的核心情绪或梗

转换示例：

输入占位符： {{media: emoji, 猫猫震惊.jpg}} 
`输出标签：` `[emj: 猫猫震惊.jpg]` 

## 6. 模型

使用轻量决策模型或语言模型。任务复杂度低（润色描述文字），不消耗主力模型资源。

## 7. 使用场景

本 Skill 不独立触发，作为所有内容生成 Skill 的通用后处理步骤。任何输出文本中包含  {{media: ...}}  占位符的 Skill，在文本进入数据库或发布前，自动调用本 Skill 将占位符转换为最终伪富媒体标签。

新增的内容生成 Skill 只需在输出中使用统一占位符语法，无需额外注册即可自动获得伪富媒体格式化能力。

## 8. Skill 定义文件

存放于  skills/media_generation/SKILL.md ，格式与其他 Skill 一致。

## 9. 优势

- 内容生成 Skill 无需学习伪富媒体标签的写法规范
- 所有伪富媒体的格式、风格、长度由一处控制
- 后续若引入真实图片生成，只需改造此 Skill 的内部逻辑，不影响其他模块
- 调优 Prompt 时只需修改一个文件
