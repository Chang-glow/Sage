# 06-2 配置管理

## 1. 概述

项目配置分为两类：敏感信息（API Key、密码）通过 .env 管理；游戏参数（遗忘速度、心流概率、等级阈值等）通过 config.yaml 管理。两者严格分离，.env 不提交 Git，config.yaml 纳入版本控制。

## 2. 配置文件结构

```
sage/
├── .env                  # 敏感配置（不提交 Git）
├── .env.example          # .env 模板（提交 Git，含空值示例）
├── config.yaml           # 游戏参数配置（提交 Git）
├── prompts/              # Prompt 模板目录
│   └── ...
└── world/                # 平陵市设定数据
    └── ...
```

## 3. 敏感配置：.env

### 3.1 .env 内容

```
# 数据库
DATABASE_URL=postgresql://user:password@localhost:5432/sga

# DeepSeek API
DEEPSEEK_API_KEY=sk-xxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com

# 硅基流动 API
SILICONFLOW_API_KEY=sk-xxxxxxxx
SILICONFLOW_BASE_URL=https://api.siliconflow.cn

# 管理员密码（观赏后台）
ADMIN_PASSWORD=xxxxxxxx

# 可选：Bing Search API
BING_SEARCH_API_KEY=xxxxxxxx
```

### 3.2 加载方式

使用 python-dotenv 在应用启动时加载：

```
from dotenv import load_dotenv
load_dotenv()
```

## 4. 游戏参数配置：config.yaml

### 4.1 可配置项

```
# 记忆系统
memory:
  decay_rate: 0.01                # 每日亲密度衰减
  max_short_fragments: 150        # 短期碎片上限
  max_long_fragments: 50          # 长期碎片上限
  short_retention_days_low: 3     # 低重要度保留天数
  short_retention_days_mid: 14    # 中等重要度保留天数
  long_retention_days: 90         # 长期保留天数

# 心流系统
flow:
  interactive_trigger_similarity: 0.8   # 互动型心流触发相似度
  spontaneous_trigger_intensity: 0.7    # 自发性心流触发强度
  max_rounds: 10                        # 单次最大轮数
  max_per_day: 2                        # 每日最大次数

# 调度系统
scheduler:
  scan_interval_minutes: 60       # 扫描间隔
  max_concurrent_agents: 5        # 最大并发 Agent

# 安全
security:
  invite_code_required: true
  max_agents_per_ip_per_day: 3
  max_agents_per_day_global: 20
  default_agent_token_limit: 20000
  global_token_limit: 1000000

# 等级
level:
  total_cap: 50                   # Agent 总数上限
  replenish_threshold_ratio: 0.9  # 补充阈值比例
  replenish_amount_min: 1
  replenish_amount_max: 3
  replenish_interval_days: 7

# 回复决策权重
reply_weights:
  base_personality: 0.4
  offline_life: 0.25
  observed_info: 0.1
  post_content: 0.25
```

### 4.2 加载方式

启动时读取 config.yaml，存入全局 Config 对象：

```
import yaml
with open("config.yaml") as f:
    config = yaml.safe_load(f)
```

## 5. Prompt 模板管理

### 5.1 存储位置

所有 Prompt 模板存放在 prompts/ 目录，按模块分文件。Prompt 模板本身在对应的 skills/ 目录的 SKILL.md 中定义（见  [06-1_Skill与插件架构](./06-1_Skill与插件架构.md) ）。

### 5.2 变量插值

Prompt 模板中的变量在运行时由系统注入：

- {agent_name}  → Agent 网名
- {active_persona}  → 当前临时激活人格面描述
- {post_title}  → 帖子标题
- {relationship}  → 与帖子作者的关系

变量插值由 Skill 引擎完成，不依赖外部模板库。

## 6. 配置验证

启动时验证所有必填配置项是否存在，类型是否正确。缺少必填项或格式错误时拒绝启动，输出错误信息。
