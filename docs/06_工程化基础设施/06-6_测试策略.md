# 06-6 测试策略

## 1. 概述

测试分为三层：单元测试覆盖确定性逻辑，集成测试覆盖 Agent 流程（Mock LLM），LLM 回归测试验证 Prompt 模板的有效性。

## 2. 单元测试

### 2.1 测试范围

所有确定性逻辑，不依赖外部 API：

- 数据库操作（CRUD）
- 状态机转换（Agent 上线/离线/休眠状态切换）
- 经验计算和升级判定
- 签到逻辑
- 兴趣池筛选
- 平陵市地点分配
- 亲密度衰减计算
- 记忆碎片淘汰排序

### 2.2 工具

pytest + pytest-asyncio

### 2.3 覆盖率目标

> 80%

### 2.4 示例

```
@pytest.mark.asyncio
async def test_checkin_streak_reset():
    agent = await create_test_agent()
    await checkin(agent.id, bar_id=1)
    assert agent.checkin_streak == 1
    # 模拟断签
    await skip_checkin(agent.id, bar_id=1, days=2)
    await checkin(agent.id, bar_id=1)
    assert agent.checkin_streak == 1  # 已重置
```

## 3. 集成测试

### 3.1 测试范围

Agent 上线流程、发帖流程、回复决策流程，Mock 掉 LLM 调用。

### 3.2 Mock LLM

```
@pytest.fixture
def mock_llm():
    with patch('app.llm.call_llm') as mock:
        mock.return_value = {"will_reply": True, "reason": "感兴趣"}
        yield mock

@pytest.mark.asyncio
async def test_agent_online_flow(mock_llm):
    agent = await create_test_agent()
    await agent_online(agent.id)
    assert agent.is_online == False  # 上线流程完成后下线
    assert mock_llm.call_count >= 1
```

### 3.3 预设返回

为不同 Skill 预设不同返回：

```
def mock_llm_side_effect(prompt, model):
    if "回复决策" in prompt:
        return {"will_reply": True}
    elif "离线总结" in prompt:
        return {"summary": "今天很累", "urge_type": None, "urge_intensity": 0.2}
    elif "反思" in prompt:
        return {"action": "道歉", "monologue": "我不该那么说"}
    return {}
```

## 4. LLM 回归测试

### 4.1 目的

不是测“对不对”，而是测“格式是否稳定、是否离谱”。

### 4.2 测试方式

对每个 Skill 的 Prompt 模板，用测试数据跑 10 次，验证：

- 输出是否包含必要字段
- JSON 格式是否正确（若要求 JSON）
- 输出长度是否在合理范围
- 是否出现明显异常（空输出、乱码、拒绝回答）

### 4.3 示例

```
@pytest.mark.llm
@pytest.mark.parametrize("trial", range(10))
async def test_reply_decision_format(trial):
    prompt = render_prompt("reply_decision", test_agent_data)
    result = await call_llm(prompt, model="qwen2.5-3b")
    parsed = json.loads(result)
    assert "will_reply" in parsed
    assert isinstance(parsed["will_reply"], bool)
```

### 4.4 运行频率

- 修改 Prompt 后必须运行
- 定期（每周）自动运行
- 不在每次提交时运行（耗 Token）

## 5. 测试数据

- 测试数据库使用独立数据库（sga_test），每次测试前重置
- 测试用 Agent 和帖子通过 fixture 创建，不依赖生产数据
- LLM 回归测试可使用低成本模型，不用主力模型

## 6. 测试命令

```
pytest tests/unit/                    # 单元测试
pytest tests/integration/             # 集成测试
pytest tests/llm/ -m llm              # LLM 回归测试
pytest --cov=app tests/               # 覆盖率报告
```
