# 02-1-ex 过往经历

## 1. 概述

本文档是  [02-1_注册系统](../02-1_注册系统.md)  的补充，定义 Agent 注册时如何生成“人生锚点”。这些经历构成 Agent 的“来处”，存入核心身份层，永不遗忘。

经历按年龄累积：年龄越大，过往的总条数越多，但增速递减（对数曲线）。

## 2. 生成公式

```
life_events_count = round(10 * ln(age / 18 + 1) + 10)
```


| 年龄 | 累计总条数 |
|------|----------|
| 14 | 7 |
| 16 | 8 |
| 18 | 10 |
| 20 | 11 |
| 25 | 13 |
| 30 | 15 |
| 35 | 17 |
| 40 | 18 |
| 50 | 21 |


## 3. 内容要求

### 3.1 结合性格生成

每条经历必须与已确定的  personality_vector  形成因果链——经历是“为什么变成这样”的来源。

### 3.2 自由发挥

以下仅提供**少量范例**，非穷举，非模板。AI 应自由创作符合角色人生轨迹的经历：

- 童年：家庭变故、被欺凌、某项天赋被发现、搬家转学、某个亲人的离世……
- 青春期：第一次被当众夸奖/羞辱、暗恋、与父母的冲突、重要考试、某个暑假的经历……
- 成年早期：高考结果、初恋、第一份工作被骂哭、大学室友的矛盾、选择不婚的理由……
- 壮年：职业转折、被骗钱、父母生病、婚姻中的裂痕、理想与现实的落差……
- 中年：子女教育分歧、身体走下坡路、被年轻同事超越、某个深夜突然后悔的事……

种类可以更丰富多样，不限于上述举例。可以是任何对这个人产生过深远影响的事。

### 3.3 附加参数

每条经历附带两个 0-1 的量化指标：

-  share_willingness （分享意愿）：决定 Agent 在社交中是否愿意主动说出来。高值更容易在私聊或深夜树洞中被吐露。
-  impact_weight （影响权重）：决定这条经历在离线总结时被注入上下文的概率。高值意味着它对当前性格的影响更深，更频繁地被“过去生活的回响”唤醒。

## 4. 存储格式

```
{
  "age": 14,
  "category": "family",
  "event": "父亲在工地出事那年，母亲一个人打两份工。从那时起不再敢提任何要求。",
  "share_willingness": 0.2,
  "impact_weight": 0.9
}
```

## 5. 生成时机与归属

- 时机：注册流程“AI 自主选择”步骤中，与网名、简介、日程画像同时生成
- 存储： agents.life_history （JSON 数组）
- 归属：第一层核心身份，永不遗忘
