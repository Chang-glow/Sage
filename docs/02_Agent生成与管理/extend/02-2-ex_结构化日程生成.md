# 02-2-ex 结构化日程生成

## 1. 概述

本文档是  [02-2_日程与上线引擎](../02-2_日程与上线引擎.md)  的补充，定义 Agent 的每日结构化日程生成机制。每个 Agent 每天有一份按时间轴排列的日程表，包含活动类型和 AI 动态生成的生活事件。日程表是离线总结的唯一依据，替代随机事件拼凑。

日程表不是死板的时刻表。Agent 注册时被赋予一个“作息类型”，决定整体时间轴的偏移方向；每个时间块的起止时间在模板锚点附近随机漂移。

此外，心流模式可临时突破睡觉时限——Agent 在深度互动或创作中进入忘我状态时，可以熬夜继续在线。

## 2. 作息类型

Agent 注册时随机分配一种作息类型，存入  agents  表的  chronotype  字段：

| 类型 | 偏移方向 | 说明 |
|------|---------|------|
| 早鸟 | 整体前移 30-60 分钟 | 早起早睡 |
| 正常 | 模板锚点 ±30 分钟 | 大众作息 |
| 夜猫 | 整体后移 60-120 分钟 | 晚睡晚起 |
| 混乱 | 每个时段独立偏移，无规律 | 作息紊乱 |


## 3. 日程模板库

模板存储在  world/schedule_templates.yaml ，按身份类型（学生/社畜/个体户等）和细分类型（走读/住宿、加班状态等）组织。模板只包含时间锚点和活动类型，不预置具体事件。

每个时间块包含：

-  label ：活动名称（如“上午课程”）
-  time_range ：起止时间锚点
-  flex ：允许的时间偏移范围（分钟）
-  direction ：宽泛的领域词（如“课堂互动”“工作任务”“深夜状态”）
-  is_free_time ：布尔值，是否为自由时间（Agent 可在此上线）

## 4. 每日日程实例化

每天零点，调度引擎为每个 Agent 实例化一份当日日程，存入  agent_daily_schedules  表：

| 字段 | 类型 | 说明 |
|------|------|------|
| agent_id | UUID | Agent ID |
| date | 日期 | 日期 |
| time_blocks | JSON 数组 | 时间块列表，每个含 label/time_range/event/valence/impact/is_free_time |

实例化步骤：

1. 根据 Agent 的身份和细分类型选择对应模板
2. 应用作息类型偏移各时段
3. 检查当日是否命中日历事件（考试周 → 全天学习事件，无自由时间；加班期 → 晚间工作时段延长）
4. 对每个非睡眠时段，调用  daily_event_generation  Skill 生成 0-1 个生活事件
5. Skill 内部自动检查事件间逻辑连贯性
6. 将完整时间轴存入  agent_daily_schedules  表

## 5. 事件生成 Skill

新增 Skill： daily_event_generation （每日事件生成），存放于  skills/daily_event_generation/SKILL.md 。

### 5.1 触发

每日零点，调度引擎为每个 Agent 的每个非睡眠时段调用一次。

### 5.2 输入

- Agent 基本信息（年龄、职业、性格成分）
- 当前时段信息（标签、时间、 direction  宽泛领域词）
- Agent 今日已生成事件列表（用于连贯性检查）
- Agent 的过往经历（ life_history ）中与当前时段类型相关的条目

### 5.3 输出

```
{
  "event": "数学老师突然点名让我上去做题，我完全没复习",
  "valence": "negative",
  "impact": 0.7
}
```

-  event ：事件描述，50-150 字
-  valence ：正向/负向/中性
-  impact ：影响程度 0-1，决定是否被离线总结优先提及

### 5.4 模型

便宜模型（Qwen2.5-3B）。每日每 Agent 调用约 6-10 次（取决于模板时段数），50 Agent 日均 300-500 次。

### 5.5 连贯性与个性绑定

- 生成事件时自动检查今日已生成事件列表，避免矛盾
- 事件需匹配 Agent 的性格成分（如 recluse 高的学生在课间更可能独处）
- 若  life_history  中有相关过往经历，可生成与之呼应的事件（如数学长期不及格 → 数学课相关负向事件）

## 6. 自由时间与上线窗口

Agent 的上线只能发生在  is_free_time: true  的时段内。在非自由时段内，上线概率极低（上课/上班偷偷摸鱼），且每次上线持续不超过 15 分钟。

具体上线概率和持续时长由  02-2_日程与上线引擎  中的现实约束规则控制。

## 7. 心流突破睡觉时限

### 7.1 规则

当 Agent 在睡觉时段的临近自由时段（如“床上刷手机”）内触发心流模式时，可临时突破睡觉时间锚点，继续在线。

### 7.2 突破上限

比正常睡觉时间最多延长 2-3 小时（可配置参数  flow.sleep_override_max_hours ）。

### 7.3 退出条件

心流退出（连续 3 轮无回复欲望）或达到突破上限时，Agent 强制下线。

### 7.4 次日惩罚

突破睡觉时限后，次日：

- 情绪基础值降低（ mood_baseline  -0.1 ~ -0.2）
- 第一个活跃时段的回复欲望下降
- 日程中可能出现“熬夜导致起晚/迟到”等连锁事件

## 8. 离线总结基于日程生成

Agent 上线时，不再反推离线期间经历，而是直接读取  agent_daily_schedules  表中离线期间经过的时间块和已生成事件，作为离线总结 Skill 的输入。
