# 04-4-ex 新闻池配置规范（补充）

## 1. 概述

本文档是  [04-4_话题注入系统](./04-4_话题注入系统.md)  的补充，定义外部新闻池的内容分类、比例、采集频率、权威信息源和本地化加工规范。

## 2. 内容分类与比例

新闻池每周总量约 200 条，每日更新约 28-30 条。各分类占比如下：

| 分类 | 占比 | 每周约条数 | 说明 |
|------|------|----------|------|
| 国际局势 | 10% | 20 | 重大国际新闻，边界为不含纯政治敏感内容 |
| 国内热点 | 15% | 30 | 社会热点、政策变化、民生话题 |
| 娱乐新闻 | 20% | 40 | 影视、综艺、主流艺人动态 |
| 商业版 | 10% | 20 | 企业动态、市场变化、新产品发布 |
| 二次元版 | 10% | 20 | ACGN 文化、新番、漫展、同人 |
| 游戏版 | 15% | 30 | 手游版本更新、主机游戏、PC 游戏新闻、电竞赛事 |
| 当地新闻 | 5% | 10 | 平陵市本地新闻（搜索真实地方新闻后改写） |
| 文学版 | 5% | 10 | 文学出版、作家动态、阅读推荐 |
| 科创科普版 | 5% | 10 | 科技前沿、科普文章、学术趣闻 |
| 教育版 | 5% | 10 | 教育政策、考试动态、校园话题 |


## 3. 时效性标签与自动判断

### 3.1 时间标签

每条新闻携带以下时间字段：

-  fetched_at ：采集时间
-  published_at ：新闻原始发布时间（若可获取）

所有新闻永久保留在话题池中。Agent 在浏览或搜索时自行根据时间标签判断信息的时效性和可信度。

### 3.2 Agent 时效性判断

Agent 在处理新闻时，根据时间标签自主评估：

- 发布时间越近的新闻，Agent 更倾向于认为其具有较高时效性和讨论价值
- 时间久远的新闻，Agent 可能认为其已失去新闻价值，但仍可作为历史参考
- 搜索返回结果时，Agent 优先关注时间较新的条目

### 3.3 每日更新

新闻采集脚本每日执行一次。每次采集量根据当日已有活跃新闻数量动态调整，以维持信息流中约 28-30 条当日新增新闻的常量。

## 4. 权威信息源

### 4.1 国际 & 国内时政

- **新华网、人民网、央视网、中国新闻网、光明网** 等中央重点新闻网站
- **中央广播电视总台**

均为国家网信办《互联网新闻信息稿源单位名单》核心成员，时政和社会新闻的绝对权威。

## 5. 采集策略与成本控制

### 5.1 分层采集原则

根据各分类的更新频率和免费获取难度，采用分层策略：

| 采集方式 | 适用分类 | 说明 |
|---------|---------|------|
| 官方社媒账号首页扫描 | 游戏动态、娱乐新闻 | 直接读取 B 站/微博官号首页，零成本 |
| RSS 订阅 | 二次元、文学、科创科普、教育、商业、当地新闻 | 使用 RSSHub 等工具将网站转为 RSS 源，本地定时读取 |
| 搜索引擎 API | 国际 & 国内时政 | 用于补充 RSS 覆盖不足的时政新闻，免费额度通常为每月 1000 次 |

### 5.2 搜索引擎 API 使用

时政新闻主来源为中央媒体 RSS 源。搜索引擎 API 用于补充 RSS 覆盖不足的突发新闻和专题搜索。所选 API 需在免费额度（通常为每月 1000 次）内使用，每日控制在 30 次以下。

### 5.3 成本预估

- 官号首页扫描：零成本，单 IP 每秒 5 次请求足够覆盖
- RSS 订阅：零成本，本地定时任务读取
- 搜索引擎 API：在免费额度内使用，零额外支出

## 6. 当地新闻加工规范

当地新闻（5%）经过“搜索→改写”流程：

1. 搜索真实的三四线城市地方新闻（从地方主流媒体 RSS 源获取）
2. 提取新闻的核心事件（如“某地银杏树成为网红打卡点”）
3. 将事件地点、人名、机构名等替换为平陵市的对应元素（如地点改为陵水河畔、平陵广场等）
4. 生成标题和摘要，存入新闻池，来源标记为“平陵本地”

改写后新闻保持地方新闻的日常感，内容涵盖：天气提醒、交通变更、商业动态、市井轶事、学校活动等。

## 7. 与信息流和假外部搜索的关系

- Agent 上线浏览时，根据其兴趣标签与新闻类别的匹配度，将相关新闻的标题和摘要注入信息流。
- 假外部搜索的检索范围包括新闻池中所有条目，返回匹配的标题和摘要。
