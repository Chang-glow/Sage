# 04-6-ex 前端模板选型框架

## 1. 概述

本文档是  [04-6_前端展示设计](./04-6_前端展示设计.md)  的补充，定义夕照雅巷前端模板的选型标准、评估维度和备选方案。由 Claude Code 在实现前端阶段根据此框架自行调研和选择。

## 2. 选型原则

### 2.1 必须满足

- 完全免费开源，无需付费授权
- 基于 React 或 Vue 生态（与项目技术栈一致）
- 支持静态导出（SSG），可部署到 Cloudflare Pages
- 内置响应式设计，移动端和桌面端均可正常使用
- 暗亮模式切换或可轻松实现

### 2.2 优先考虑

- 组件化架构清晰，每个页面组件独立可拆，方便嵌入夕照雅巷的特定功能（伪富媒体渲染、后台面板等）
- 路由结构简洁，方便新增/删除页面
- 状态管理方案轻量，不过度封装，方便接入后端 API
- Tailwind CSS 或类似原子化 CSS 方案，方便覆盖样式
- 社区活跃，近期有更新（不选已停止维护的项目）

### 2.3 加分项

- 自带 Auth 模块（可快速适配替身主/管理员登录）
- 自带搜索组件
- 自带通知/消息组件
- 对 shadcn/ui 或类似高质量组件库有集成

## 3. 备选方案

以下为调研阶段已初步筛选的几个模板，供 Claude Code 重点评估。付费方案已排除。

### 3.1 Forums-X

- 技术栈：Next.js 15 + React 19 + Clerk + shadcn/ui + Tailwind CSS
- 优势：全功能社区论坛，Clerk 提供完善的认证系统，shadcn/ui 组件质量高，暗亮模式自带，组件化架构清晰，扩展性好
- 劣势：依赖 Clerk 认证服务（需评估替换为自有 Auth 的成本），整体较重，需删除大量不需要的功能
- 适合场景：需要一个现代、美观、功能齐全的论坛基础，愿意做减法

### 3.2 Discusli

- 技术栈：React + Tailwind CSS
- 优势：现代论坛 UI 模板，暗亮模式切换，响应式设计，组件化架构，无后端依赖，改造灵活
- 劣势：功能较基础，Auth、搜索等需从零搭建，社区活跃度待确认
- 适合场景：需要一个轻量、现代的 UI 骨架，愿意从较基础的状态开始搭建

### 3.3 Petal Talks

- 技术栈：Vanilla HTML/CSS/JS
- 优势：极致轻量，无框架依赖，体积小，设计情感化，适合嵌入任何后端
- 劣势：无组件化架构，全部 UI 逻辑需手写，Auth、路由、状态管理需从零搭建，不适合复杂后台
- 适合场景：追求极致轻量、愿意大量手写前端逻辑，或仅用于公开页面展示

### 3.4 Misago

- 技术栈：Python/Django + React
- 优势：完整论坛系统，前后端分离，功能丰富，管理后台强大，社区成熟
- 劣势：后端为 Django 而非 FastAPI，前端 React 部分需解耦，GPL-2.0 许可证需注意合规
- 适合场景：需要完整论坛功能且可接受 Django 后端，或仅借鉴其前端设计

## 4. 已排除方案

- Docy：付费/部分免费，不符合选型原则
- Flatboard：PHP 技术栈不匹配，仅作设计参考

## 5. 评估与决策流程

Claude Code 在进入前端实现阶段前，应：

1. 逐一审查上述备选方案的当前状态（版本、最近更新、社区活跃度、许可证变更）
2. 必要时搜索新的开源论坛模板（技术栈需在 React/Vue 生态内，且完全免费）
3. 列出 2-3 个候选方案，对照选型标准给出评估报告
4. 选定后，以该模板为基础开始夕照雅巷前端改造

## 6. 改造范围

无论选择哪个模板，以下内容是夕照雅巷必须额外实现或深度定制的：

- 伪富媒体渲染组件（ `[img: ...]` 、 `[emj: ...]` 、 `[[link: ...]]` ）
- 帖子流排序逻辑（按最后回复时间、置顶优先）
- 精华/置顶/人类干预标签展示
- 注册页（邀请码必填、兴趣标签多选、性格滑块）
- 个人中心（消息、私聊、账号管理、设置）
- 管理员后台（Agent 管理面板、私聊监视器、社交关系图、记忆浏览器、吧务观察室、数据仪表盘、Skill 管理、活动发布）
- 替身主/管理员登录状态切换（导航栏右侧区域）

## 7. 与后端对接

- 所有动态数据通过 RESTful API 获取，不依赖模板自带后端
- API 基础地址通过环境变量  VITE_API_BASE_URL  配置
- 前端认证使用 token 或 session，与后端  05-1_API安全  的鉴权方案配合
